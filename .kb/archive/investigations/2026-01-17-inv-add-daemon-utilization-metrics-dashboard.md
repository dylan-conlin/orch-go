## Findings

### Finding 1: Daemon spawn events are tracked separately from session.spawned

**Evidence:** `daemon.spawn` events are emitted in daemon.go:450 when the daemon spawns an agent. `session.spawned` events are emitted in spawn_cmd.go for all spawns (manual and daemon).

**Source:** `cmd/orch/daemon.go:450`, `cmd/orch/spawn_cmd.go:1472`

**Significance:** Can compute daemon spawn rate by counting `daemon.spawn` events divided by `session.spawned` events.

---

### Finding 2: Triage bypass tracking already exists

**Evidence:** `spawn.triage_bypassed` events are emitted when users explicitly use `--bypass-triage` flag. This is logged in spawn_cmd.go:2233.

**Source:** `cmd/orch/spawn_cmd.go:2228-2240`

**Significance:** Can track triage slip rate (bypasses / total spawns) to surface where triage discipline is slipping.

---

### Finding 3: Events.jsonl aggregation pattern exists in stats_cmd.go

**Evidence:** The `orch stats` command aggregates events.jsonl with time-window filtering and rate calculations. The same pattern was reused for utilization metrics.

**Source:** `cmd/orch/stats_cmd.go:248-284` (parseEvents), `cmd/orch/stats_cmd.go:286-913` (aggregateStats)

**Significance:** Provided a proven pattern for computing metrics from events.jsonl with configurable time windows.
