# Investigation: Flip Default Spawn Mode Headless

**Question:** How is the default spawn mode currently implemented, and what changes are needed to flip from tmux to headless?

## Findings

### Finding 1: Spawn mode logic uses flag priority with tmux as fallback

**Evidence:** In runSpawnWithSkill, the logic checks inline flag first, then headless flag, then defaults to tmux mode (lines 1032-1043 in cmd/orch/main.go).

**Source:** cmd/orch/main.go:1032-1043

**Significance:** To flip default to headless, we need to invert this logic and add a --tmux flag for opt-in.

---

### Finding 2: Three spawn mode flags currently exist

**Evidence:** spawnInline (line 162), spawnHeadless (line 163), spawnAttach (line 164). Each has corresponding flag registration (lines 236-238).

**Source:** cmd/orch/main.go:162-164, 236-238

**Significance:** Need to add a fourth flag (spawnTmux) to enable explicit tmux opt-in.

---

### Finding 3: Help text explicitly states "Default: tmux window"

**Evidence:** Line 207 says "orch-go spawn investigation \"explore the codebase\" # Default: tmux window"

**Source:** cmd/orch/main.go:207

**Significance:** Documentation needs updating to reflect new default (headless) and show --tmux as opt-in.

---

## Synthesis

**Key Insights:**

1. **Flag-based mode selection** - Spawn mode is determined by boolean flags in priority order, making it straightforward to add --tmux and adjust the default logic.

2. **Minimal surface area** - Changes are localized to flag declarations, runSpawnWithSkill logic, and help text - no protocol or API changes needed.

3. **Backward compatibility via explicit opt-in** - Adding --tmux flag allows users to preserve tmux behavior while changing the default to headless.

**Answer to Investigation Question:**

The default spawn mode is implemented via cascading if-else checks (inline → headless → tmux as fallback) in runSpawnWithSkill. To flip the default to headless, we need to: (1) add a --tmux flag, (2) reverse the logic to check tmux flag first, then default to headless if none specified, (3) update help text and flag descriptions.
