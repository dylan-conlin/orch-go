## Findings

### Finding 1: Agent counts were duplicate in stats bar

**Evidence:** The stats bar at lines 278-311 in +page.svelte showed active/recent/archive counts, but these same counts were already visible in the collapsible section headers at lines 462-509.

**Source:** web/src/routes/+page.svelte:278-311, web/src/routes/+page.svelte:462-509

**Significance:** Removing duplicates frees space for more valuable information (usage limits).

---

### Finding 2: Usage API needed reset times

**Evidence:** The `/api/usage` endpoint returned utilization percentages but not reset times. The Go `usage.UsageLimit` struct already had `TimeUntilReset()` method but it wasn't exposed.

**Source:** cmd/orch/serve.go:707-715, pkg/usage/usage.go:47-69

**Significance:** Reset times are essential for users to know when limits will refresh.

---

### Finding 3: Progress bar implementation uses Tailwind classes

**Evidence:** Used Tailwind utility classes for progress bars: `h-2 w-20 rounded-full bg-muted` for container, conditional `bg-green-500/bg-yellow-500/bg-red-500` for fill based on percentage thresholds.

**Source:** web/src/routes/+page.svelte:281-320

**Significance:** Consistent with existing dashboard styling, color-coded for quick visual assessment.
