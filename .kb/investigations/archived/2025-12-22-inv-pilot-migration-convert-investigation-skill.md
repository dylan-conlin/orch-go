## Implementation Recommendations

**Purpose:** Document the pattern for future skill migrations.

### Recommended Approach ⭐

**skill.yaml structure for SKILL.md compilation:**

```yaml
name: skill-name
version: 1.0.0
description: Skill description
type: skill           # Required for SKILL.md output
output: SKILL.md      # Output filename
frontmatter: frontmatter.yaml  # YAML frontmatter file
sources:
  - intro.md
  - workflow.md
  - ...
```

**Why this approach:**
- `type: skill` tells skillc to emit frontmatter first
- `frontmatter: frontmatter.yaml` keeps frontmatter separate from content
- Sources listed in order they should appear

**Implementation sequence:**
1. Create .skillc/ directory
2. Create frontmatter.yaml with skill metadata
3. Create source files (intro.md, etc.)
4. Create skill.yaml manifest
5. Run `skillc build`
6. Verify frontmatter at line 1
7. Backup old SKILL.md
8. Test skill loading
