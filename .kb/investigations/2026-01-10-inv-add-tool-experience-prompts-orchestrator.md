1. **Skill architecture varies by complexity** - Investigation skill uses modular approach (5 source files) making it easy to add sections, while orchestrator skill uses monolithic approach (1 source file). Both are valid patterns, but modular scales better for complex skills.

2. **Strategic placement matters** - Tool experience prompts placed before TEST-FIRST GATE in investigation skill and after Orchestrator Autonomy in orchestrator skill. This ensures agents ask about Dylan's experience before diving into elaborate work.

3. **Addresses trust calibration loop** - Both changes directly address the trust calibration meta-pattern where Dylan defers to AI despite having relevant tool experience. The system now prompts for this context instead of assuming.
