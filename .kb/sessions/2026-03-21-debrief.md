## What We Learned

This session discovered something the 22 daemon research tasks and 777 orphaned investigations never could — because the discovery required a human feeling directionless and asking why.

**Three failure modes stack from cognitive to systemic:** (1) The **recognition trap** — skimming agent output feels like understanding but changes nothing in your mental model. (2) **Delegation gravity** — thinking is hard, so you gradually cede judgment to the system, each step rational, cumulative effect is nobody's driving. (3) The **improvement loop** — unlike entropy spirals which break things visibly, governance accretes invisibly. 35% of the codebase watching the other 65%, 22 of 26 daemon tasks producing zero actionable issues.

**The autonomous knowledge pipeline was disconnected from agent work.** 253 probes exist but none came from the daemon's ClaimProbeGeneration. Zero model drift issues, zero tension cluster issues ever created. Models get maintained by workers at completion time. The 15 research daemon tasks ran every cycle producing nothing.

**The key architectural insight:** as long as the orchestrator can bypass the daemon via `orch spawn`, the daemon never experiences the friction it needs to improve as an attractor. This is "pressure over compensation" applied to the spawn pipeline. New design: orchestrator's only output is well-scoped beads issues, daemon is sole executor. This forces scoping quality, separates judgment from execution, and routes all friction where it can drive improvement.

**The deepest irony:** the system designed to detect emergent coordination failures among amnesiac agents was itself an emergent coordination failure among amnesiac agents. "Gates producing gates" (Layer 4) was already happening — organically, one justified commit at a time.
