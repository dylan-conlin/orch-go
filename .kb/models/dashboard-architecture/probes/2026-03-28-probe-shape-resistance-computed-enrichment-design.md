# Probe: Shape/Resistance as Computed Enrichment — Not Standing Elements

**Date:** 2026-03-28
**Model:** dashboard-architecture
**Investigation:** `.kb/investigations/2026-03-28-design-expand-product-surface-shape-resistance.md`

## Question

How do shape classification (search/convergence/forming/execution) and resistance signal (bolted-on vs just-hard) integrate into the three-element product surface?

## Finding

**Verdict: EXTENDS** — Shape and resistance are a new content type (computed insight) that flows through existing comprehension elements, not new standing elements. This extends invariant 10's rendering mode taxonomy from three modes (content, metadata, presence) to four.

### Evidence

1. The converged five-elements thread rejected five standing elements because shape is momentary and resistance is episodic — neither has the persistent temporal quality of threads/briefs/tensions.

2. But both signals pass the product test (metadata → content rendering): shape classification currently exists as invisible routing, resistance exists as raw metrics. Making them visible IS product clarification.

3. The resolution: computed enrichments. Shape surfaces as one-line annotations in orient output and thread entries. Resistance surfaces as computed tension entries in "What's still open" when an area shows a bolted-on pattern.

4. All data already exists: routing inference (shape), entropy fix:feat (resistance), beads issue chains (resistance), area labels (resistance), orient throughput (shape).

### Model Impact

- **Invariant 10 update:** Four rendering modes, not three — content, metadata, presence, and **computed insight** (signals derived from data the user can't see, rendered as natural language within existing comprehension elements).
- **New mechanism:** Computed tension generation — the system can produce tension entries (appearing in "What's still open") from data analysis, not just from human-created questions or thread states.
- **Evolution note:** The home surface stays at three elements but gains a fourth content generation mechanism alongside human-authored threads, agent-produced briefs, and human-created questions.

### Falsifiability

- If shape classification accuracy is below ~80%, it should remain orchestrator judgment
- If computed tensions feel robotic alongside human-written tensions, they may need a different rendering surface
- If Dylan ignores shape/resistance annotations for 2+ weeks, the signal value hypothesis is wrong
