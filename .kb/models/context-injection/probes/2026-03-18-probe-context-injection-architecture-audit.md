# Probe: Context Injection Architecture — 2-Month Drift Audit

**Date:** 2026-03-18
**Parent Model:** context-injection
**Trigger:** Knowledge decay alert (999d since last probe — model created Jan 17, never probed)
**Method:** Compare model claims against current codebase and configuration

## Claims Tested

### Claim 1: Hook Files — `session-start.sh`, `load-orchestration-context.py`, `bd prime`

**Model says:** "Key Files: `session-start.sh`, `load-orchestration-context.py`, `bd prime`" for Claude Code hook-based injection.

**Current state:**
- `~/.claude/session-start.sh` — **DOES NOT EXIST** (deleted)
- `~/.claude/load-orchestration-context.py` — **DOES NOT EXIST** (deleted)
- Current SessionStart hooks are:
  1. `~/.claude/hooks/load-skill-identity.sh` — loads orchestrator skill for interactive sessions
  2. `~/.claude/hooks/orient-hook.sh` — calls `orch orient --hook` for session orientation

**Verdict:** **STALE.** The original Python-based context loading pipeline has been completely replaced by two simpler bash hooks that delegate to compiled Go (`orch orient`). The hook architecture shifted from "Python script assembles context" to "orch binary generates context."

### Claim 2: OpenCode Plugins for Execution Sessions

**Model says:** "OpenCode Plugins (Execution Sessions) — JS/TS plugins that reference local files (~4KB)."

**Current state:** Since Feb 19, 2026, Anthropic banned subscription OAuth in third-party tools. All Anthropic model work now routes through Claude CLI (tmux), not OpenCode. OpenCode is only used for non-Anthropic models.

**Verdict:** **STALE.** OpenCode plugin injection path exists but is no longer the primary execution path. Claude CLI with SPAWN_CONTEXT.md piped via stdin is the dominant pattern. See `pkg/spawn/claude.go:142` — context is piped directly: `cat CONTEXT.md | claude --dangerously-skip-permissions`.

### Claim 3: CLAUDE_CONTEXT Role-Aware Filtering

**Model says:** Three levels — `meta-orchestrator`, `orchestrator`, `worker`.

**Current state:**
- `pkg/spawn/claude.go:142` exports `CLAUDE_CONTEXT` for every spawn
- `pkg/spawn/config.go:562` has `ClaudeContext()` method
- `~/.claude/hooks/load-skill-identity.sh` checks: skips for `worker` and `meta-orchestrator`
- `~/.claude/hooks/orient-hook.sh` same filtering logic
- Tests in `pkg/spawn/claude_test.go` verify all three levels

**Verdict:** **CONFIRMED.** CLAUDE_CONTEXT role filtering is still active and correct. The three levels are used exactly as described.

### Claim 4: Failure Mode 1 — Token Bloat from Workers Receiving Orchestrator Skill

**Model says:** Workers receive 86KB orchestrator skill due to hooks running without checking CLAUDE_CONTEXT.

**Current state:** Both SessionStart hooks explicitly check `CLAUDE_CONTEXT == worker` and exit early. The fix was implemented and is still in place.

**Verdict:** **RESOLVED.** The failure mode was fixed and the fix persists. The model should note this is now mitigated, not an active failure mode.

### Claim 5: Failure Mode 2 — Redundant Guidance (bd prime + skill + SPAWN_CONTEXT)

**Current state:** `bd prime` is not in any SessionStart hook. Skill injection only happens for orchestrator sessions. SPAWN_CONTEXT.md is the single authoritative source for spawned agents.

**Verdict:** **RESOLVED.** Deduplication was achieved by making SPAWN_CONTEXT.md authoritative for workers and limiting hooks to orchestrator-only injection.

### Claim 6: Failure Mode 3 — Session Resume Leakage

**Current state:** `session-start.sh` no longer exists. The orient hook delegates to `orch orient --hook`, which generates fresh orientation rather than blindly injecting handoffs.

**Verdict:** **RESOLVED.** The mechanism that caused this failure mode has been replaced entirely.

### Claim 7: Constraint — SPAWN_CONTEXT.md as Authoritative Source

**Current state:** `pkg/spawn/worker_template.go` defines the full SPAWN_CONTEXT template. It is generated by `pkg/spawn/context.go` and piped to Claude CLI. All spawn context flows through this single path.

**Verdict:** **CONFIRMED.** This constraint is still the architectural truth. SPAWN_CONTEXT.md generation has been consolidated into the Go spawn package.

## New Developments Not in Model

1. **skillc-compiled skills:** The skill system now uses `skillc deploy` to compile and deploy skills from `skills/src/` to `~/.claude/skills/`. Context injection for workers includes compiled skill content directly in SPAWN_CONTEXT.md (via `pkg/spawn/skill_processing.go`).

2. **KB context injection:** `pkg/spawn/kbcontext.go` provides knowledge base context injection into SPAWN_CONTEXT.md — a new injection source not mentioned in the model.

3. **Config resolution:** SPAWN_CONTEXT.md now includes a CONFIG RESOLUTION section showing backend, model, tier, and mode selections with their sources.

4. **PreToolUse hooks for governance:** The hook system has expanded beyond SessionStart. PreToolUse hooks now provide runtime governance (gate-bd-close, gate-worker-git-add-all, etc.) — a new category of context injection at tool-call time rather than session-start time.

5. **Orient hook:** `orch orient --hook` provides dynamic orientation (throughput metrics, active agents, pending work) at session start — replacing the static Python-assembled context.
