## What I Observed

### 1. Workspace Name Generation (CONFIRMED)
- **Format:** `{project-prefix}-{skill-prefix}-{task-slug}-{date}-{unique}`
- **Example:** `og-inv-analyze-spawn-workflow-15feb-1424`
  - `og` = project prefix from "orch-go" (first letter of each word for 2-part names)
  - `inv` = investigation skill prefix (from SkillTierDefaults map)
  - `analyze-spawn-workflow` = task slug generated by `generateSlug(task, 3)` which:
    - Extracts meaningful words from task description
    - Filters out stop words (articles, prepositions, question words, etc.)
    - Joins first 3 meaningful words with hyphens
  - `15feb` = date in `02Jan` format, lowercased
  - `1424` = 4-character hex unique suffix (2 random bytes) to prevent collisions

### 2. SPAWN_CONTEXT.md Structure (CONFIRMED)
- Template includes sections for:
  - `TASK:` - task description
  - `SPAWN TIER:` - light or full (determines SYNTHESIS.md requirement)
  - KB context section (injected via `{{.KBContext}}`)
  - Design reference section (for ui-design-session handoff)
  - Bug reproduction section (for bug fixes)
  - Skill content (stripped for --no-track spawns)
  - Authority delegation rules
  - Session complete protocol

### 3. KB Context Integration (CONFIRMED)
- **Tiered search strategy:**
  1. First tries local project (`kb context {query}` without --global)
  2. If sparse (<3 matches), expands to global search (`kb context --global {query}`)
  3. Post-filters global results to orch ecosystem repos
  4. Applies per-category limits to prevent context flood
- **Timeout:** 5-second timeout per query to prevent hangs from kb context scanning large directories
- **Located in:** `pkg/spawn/kbcontext.go:112-203` (`RunKBContextCheck` function)

### 4. Tier System (CONFIRMED)
- **SkillTierDefaults mapping:**
  - **Full tier** (requires SYNTHESIS.md): investigation, architect, research, codebase-audit, design-session, systematic-debugging
  - **Light tier** (no SYNTHESIS.md): feature-impl, reliability-testing, issue-creation
  - **Default:** TierFull for unknown skills (conservative)
- **Tier file:** Stored in `.orch/workspace/{workspace-name}/.tier`
- **Current workspace tier:** `full` (confirmed for investigation skill)

### 5. Beads ID Tracking (CONFIRMED)
- **Tracked spawns:** Use real beads ID from `--issue` flag or created via beads integration
- **Untracked spawns (--no-track):** Generate placeholder ID format: `{project}-untracked-{timestamp}`
  - Example: `orch-go-untracked-1771204529`
  - These IDs don't exist in beads database, so `bd comment` commands fail with "issue not found"
- **Constraint confirmed:** "Untracked spawns (--no-track) generate placeholder beads IDs that fail bd comment commands"

### 6. Session Scoping (CONFIRMED)
- **Headless spawn:** Uses HTTP API with `x-opencode-directory` header to set session directory
- **Located in:** `pkg/spawn/backends/headless.go:93-110`
- **Fix for cross-project spawns:** Explicitly passes `cfg.ProjectDir` to `CreateSession` and `SendMessageInDirectory`
- **Previously failed:** When `--workdir` differed from orchestrator's CWD, session would use orchestrator's directory
- **Model claim validation:** "Session directory is set from spawn caller's CWD, not `--workdir` target" - this is **outdated**, the fix is implemented

### 7. Spawn Workflow Sequence (CONFIRMED)
14-step workflow in `cmd/orch/spawn_cmd.go:1034-1145`:
1. Pre-flight checks
2. Resolve project directory
3. Load skill and generate workspace name
4. Setup beads tracking
5. Resolve and validate model
6. Gather spawn context (KB context, gap analysis, model detection)
7. Extract bug reproduction info
8. Build usage info
9. Determine spawn backend
10. Load design artifacts
11. Build spawn context struct
12. Build spawn config
13. Validate and write context (creates SPAWN_CONTEXT.md)
14. Dispatch spawn

### 8. Workspace Files (OBSERVED)
AGENT_MANIFEST.json contains:
- `workspace_name` - kebab-case workspace name
- `skill` - skill name
- `beads_id` - real or placeholder ID
- `project_dir` - absolute project path
- `git_baseline` - git commit SHA for verification
- `spawn_time` - ISO 8601 timestamp
- `tier` - "light" or "full"
- `spawn_mode` - "claude", "headless", or "tmux"
- `model` - full model identifier
