## The Recovery

**Burning Man timing.** Exited psychosis around Burning Man. During psychosis had stopped using rave drugs (ecstasy, acid), claiming "enlightenment" that made them feel like chains. Post-psychosis, tried them again — they felt good like they used to. The drugs weren't the issue; the frame distortion was. Most concrete proof that psychosis was a frame distortion, not genuine insight: it reorganized his relationship to *everything*, not just AI.

**The deliberate boring.** Picked a boring project at work. Raw Claude Code, no fancy CLAUDE.md. Stripped away everything he'd added. Rebuilt from ground up.

**What emerged from boring:** The investigation pattern. Cross-session learnings. Saving what you learned, loading it next time. No mysticism — just files. This was the grounded version of what psychosis had been grasping at (making Claude "learn").

**Claude Code skill system launch.** Resonated immediately because it did two things: (1) validated that the original instinct wasn't crazy — reusable patterns for AI agents was a real idea, (2) gave structure and *permission to get excited about AI again*. The permission mattered — excitement itself had become dangerous after psychosis.

## The Visionary Trap

**Somatic signal:** The visionary mindset has a physical feeling — "a giddy elation you feel in your body. You think you're onto something rare and real and only you can see it."

**The warning problem:** Added a "visionary trap" warning to global CLAUDE.md to catch the drift. But the warning created a new problem — Claude would clamp down even on legitimate curiosity about possibilities. A behavioral constraint that fires on false positives. (This is the same gate calibration problem from architectural enforcement: too strict → bypass culture or suppressed legitimate exploration.)

**The deeper pattern:** Post-psychosis, Dylan lost trust in his own judgment about what's real. External validation (skill system launch, work output, social reception) became the verification mechanism for his own ideas — same pattern as the Excel-to-SQL verification targets, but applied to his own thinking.

## The Orch-Go Origin

**Influence:** The creator of the superpowers skill project. Dylan loved the brainstorming skill and found a pattern about tmux.

**The spark:** Realizing tmux would allow orchestrating Claude Code — multiple agent sessions managed programmatically. This was the technical insight that became orch-go.

**The arc:** org-mode (manual KB) → gptel (conversational AI) → aider (delegated AI) → Claude Code (autonomous AI) → psychosis (ungrounded theory) → recovery (boring + investigations) → skill system (permission to re-engage) → superpowers tmux pattern → orch-go (orchestration infrastructure).
