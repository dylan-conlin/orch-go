# Abiogenesis parallel — how does coordination emerge from substrates that have none?

## 2026-03-10

Knowledge physics describes what happens without coordination. Abiogenesis is the origin story of coordination itself. The primordial substrate meets all five conditions (amnesiac molecular agents, shared mutable substrate, locally correct thermodynamic contributions, non-trivial composition, no coordination). Prediction holds: you get accretion (random polymerization, chemical noise). Three structural parallels: (1) Autocatalysis = first attractor — a self-catalyzing molecule pulls contributions toward itself, same as a model attracting probes or a package attracting code. Key difference: this attractor emerged FROM the substrate rather than being engineered. Digital substrates can't do this — the falsifiability probe's finding that digital substrates lack substrate-embedded coordination traces directly here. (2) Lipid membranes = first gate — hard, deterministic, compositional boundary. Life didn't start with advisory conventions. The gate came before complexity, not after. (3) Selective degradation (enzymes) = first entropy management — life solved the creation/removal asymmetry by inventing targeted removal. The knowledge equivalent would be automated pruning that knows what to remove. Possible universal sequence: attractors (cheap, emergent) → gates (expensive, structural) → active entropy management (most expensive, requires meta-coordination). This maps to both abiogenesis (autocatalysis → membranes → enzymatic regulation) and orch-go (package attractors → pre-commit gates → entropy agents). If this sequence is universal, knowledge physics describes not just degradation dynamics but developmental stages of coordination in any substrate. Open question: can digital substrates develop substrate-embedded coordination (digital stigmergy) rather than requiring all coordination to be explicitly engineered? CRDTs are one example. Could programming language type systems be another? Too speculative for the main publication but potentially powerful as a Future Directions thread or closing paragraph.

Digital stigmergy is designable, not just theoretical. Existence proofs: CRDTs (mathematical convergence guarantees — the substrate can't conflict), type systems (compiler routes code through imports — the pheromone trail is the type signature). Design spectrum: fully explicit (hooks, gates, conventions — bolted on, bypassable) → structurally coupled (package structure, directory conventions — agents follow the grain) → substrate-embedded (CRDTs, type systems — can't conflict even if you try). The design question: how far rightward can you push coordination? Concrete examples: .kb/ schema requiring model reference for investigations (like Go requiring import resolution), file creation with compositional coherence checks as write-operation properties (not bolted-on linters), built-in artifact decay (auto-archive after N days without citation — pheromone evaporation as filesystem property). Critical tradeoff: substrate-embedded coordination trades flexibility for coherence. CRDTs limit expressible operations. Type systems constrain what you can write. Pheromone trails work for path-finding but not novel exploration. This tradeoff may explain the attractor → gate → entropy management sequence — each stage trades more flexibility for more coordination, and the right level depends on where you are on the accretion curve. Worth naming in the publication: you can't 'make everything a CRDT' because the flexibility/coherence tradeoff is fundamental, not incidental.
