# Brief: orch-go-covg8

## Frame

The prior agent searched 23 names, found "Kenning" as the strongest, and recommended a two-layer architecture: product name for external surfaces, orch for internal code. The question I inherited was whether this architecture actually fits the product boundary — a product whose identity just shifted from "comprehension layer" to "thinking surface with five elements." But the deeper question turned out to be whether the product needs a name at all right now.

## Resolution

The Premise Before Solution principle caught a conflation. "Name the product" presupposes the product faces external users imminently. It doesn't — the blog post does. A blog post needs a *method* name (what you teach), not a *product* name (what you install). The product doesn't need a name until the method guide ships.

This revealed three naming layers where the prior investigation saw two. The method name faces blog readers and conference attendees. The product name faces users who want to try the tool. The code name faces developers who build with it. Each audience appears at a different moment, so each name can be a different commitment: the method name is cheap and reversible ("Kenning: compound understanding from AI agent work"), the product name is medium-cost (README rewrite, domain, packages), and the code rename is expensive (392 Go files, muscle memory, every script).

The surprise was that Kenning got *stronger*, not weaker, when tested against the five-element identity. The word means both "to know" (elements 1-3: what do I know?) and "to perceive" (elements 4-5: what should I do?). That double meaning was accidental — the prior agent chose it for the comprehension fit — but it covers the broader product identity too.

## Tension

The three-layer architecture is clean in theory, but the practical question is whether a method name and a product name actually stay separate. If the blog post lands and "Kenning" becomes the handle people use, it'll become the product name by default. The staging plan (method name now, product name later) might be a distinction that dissolves on contact with reality. Whether to lean into that convergence or keep the option open is a judgment call about how much optionality is worth at this stage.
