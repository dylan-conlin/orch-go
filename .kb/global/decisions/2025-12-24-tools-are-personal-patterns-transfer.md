# Decision: Tools Are Personal, Patterns Transfer

**Status:** Accepted
**Date:** 2025-12-24
**Context:** Ecosystem identity and sharing philosophy discussion

---

## Decision

Share patterns, not tools. Implementation is now cheap; insight is scarce.

## Context

During a discussion about sharing philosophy for the orch ecosystem, a key realization emerged:

**Before AI-assisted coding:**
- Patterns were cheap (write a blog post)
- Implementation was expensive (weeks/months)
- Share the tool, pattern comes along for free

**After AI-assisted coding:**
- Patterns are still cheap to share
- Implementation is ALSO cheap (hours/days with AI)
- The tool isn't the scarce thing anymore

## What's Actually Scarce

1. **Hard-won insight** - 1,195 issues through the system, discovering "gate over remind" because reminders failed. That learning took months of real usage.

2. **Distinction-making** - "Phase vs Status" emerged from conflation causing problems. These distinctions are empirical, not theoretical.

3. **Coherent philosophy** - The principles document represents a consistent worldview about LLM constraints. Coherence is harder to generate than code.

## Implications

**For sharing:** Share the thinking. Let people build their own tools. Their implementation will fit their workflow better anyway.

**For the tools:** orch-go is shaped by specific habits (tmux, three-window layout, beads relationship). Someone else might want VS Code integration or a web UI. The patterns transfer; the tool wouldn't fit them.

**The inversion:**
- Old model: Tool embodies pattern → users adopt tool → pattern spreads through usage
- New model: Pattern exists independently → users generate their own tool → pattern spreads through understanding

The tool becomes *evidence* the pattern works, not the *vehicle* for the pattern.

## Origin

This emerged from discussing beads - a tool that was valuable both as implementation AND as pattern source. The resolution: some tools demonstrate patterns (worth sharing as examples), but the pattern is what transfers.
