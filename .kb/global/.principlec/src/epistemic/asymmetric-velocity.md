### Asymmetric Velocity (Meta-Principle)

Systems can change faster than humans can process the change. Each processing dimension requires its own gate.

**The test:** "Is the system producing faster than I can consume meaningfully?"

**What this means:**

This is a family of principles, not a single constraint. The system can outpace human capacity in multiple dimensions:

| Dimension | What Outpaces | Human Capacity | Principle | Gate |
|-----------|---------------|----------------|-----------|------|
| Code | Changes | Behavioral verification | Verification Bottleneck | 1 verification per 3 changes |
| Observability | New visibility | Meaning interpretation | Understanding Lag | "New or newly-visible?" |
| Knowledge | AI recommendations | Relevant experience | (AI Deference Pattern) | "Do I have experience here?" |
| Models | Model/decision production | Verification against reality | (Model Drift) | "Is this model still true?" |

Each dimension requires its own gate because the failure modes differ:
- Code outpacing verification → system spirals while looking healthy
- Observability outpacing understanding → misinterpret new visibility as new problems
- Recommendations outpacing experience surfacing → defer when you shouldn't
- Models outpacing reality-checking → stale models cause wrong decisions across sessions

**Why this is foundational:** LLM-based systems can produce at machine speed. Humans process at human speed. The mismatch is structural, not incidental. Without explicit gates, unprocessed output accumulates.
