The insight here — agents naturally produce artifacts, the system's job is placement not instruction — is the supply-side of the thread-centric model. Threads are the demand-side: the composing structure where placed artifacts become understanding.

## 2026-03-24

AI agents naturally create artifacts — it's trained behavior, not convention. Without structure, this produces PLAN.md, NOTES.md, TODO.md accreting at project root, getting cleaned up, reappearing next session. The agent is doing the right thing in the wrong place.

The artifact system (investigations, workspaces, threads) is the attractor system for this natural behavior. Same pattern as coordination experiments: agents without placement put code in the gravitational default (end of file, project root). Agents with attractors put it where it composes.

The community treats agent output as disposable — run agent, get code change, discard context. This misses that agent process artifacts (what it learned, tried, decided, what's unresolved) are valuable in their own right. Agents are knowledge generators that happen to also write code.

The insight: you don't need to teach agents to create artifacts. You need to build the attractors so the artifacts they naturally create land in the right place and compose into understanding.
