## orch-go servers start

Start servers via tmuxinator (legacy)

### Synopsis

Start development servers for a project using tmuxinator.

**Note:** This is a legacy command. Consider using `orch servers up` for launchd-based
server management, which provides automatic restart and better logging.

This runs `tmuxinator start workers-{project}` which creates a tmux session
with the servers window.

```
orch servers start <project> [flags]
```

### Examples

```bash
orch servers start myproject
```

### Output

```
Starting servers for myproject...
Servers started in session: workers-myproject
Attach with: orch servers attach myproject
```

### Prerequisites

Requires a tmuxinator config at `~/.config/tmuxinator/workers-{project}.yml`.
