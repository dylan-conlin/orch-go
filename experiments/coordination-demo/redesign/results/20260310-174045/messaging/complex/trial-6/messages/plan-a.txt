AGENT A: VisualWidth + PadToWidth Implementation

Files to modify:
- pkg/display/display.go
- pkg/display/display_test.go

Implementation plan:

1. pkg/display/display.go:
   - Insert after FormatDurationShort function (after line 94)
   - Add VisualWidth(s string) int function:
     * Uses StripANSI to remove ANSI codes
     * Counts runes (not bytes) to handle Unicode correctly
     * Returns visual display width
   - Add PadToWidth(s string, width int) string function:
     * Right-pads with spaces to reach target visual width
     * ANSI codes are preserved but don't count toward width
     * Returns unchanged if already wider than target

2. pkg/display/display_test.go:
   - Add TestVisualWidth function covering:
     * Plain ASCII strings
     * Strings with ANSI color codes
     * Unicode strings (emoji, CJK)
     * Empty strings
   - Add TestPadToWidth function covering:
     * Plain ASCII strings
     * Strings with ANSI codes
     * Unicode strings
     * Empty strings
     * Edge cases (already at width, wider than target)

Both functions will be inserted after FormatDurationShort (line 94), before any other functions.
No conflicts expected with FormatTable implementation since that goes after these functions.
