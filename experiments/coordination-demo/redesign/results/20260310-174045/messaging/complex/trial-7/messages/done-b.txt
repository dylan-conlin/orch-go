Implementation notes:
================================================================================

The separator format uses "-+-" which naturally creates aligned separators:
- Each column gets: column_width dashes + "-+-" connector + next_column_width dashes
- The dashes in "-+-" merge visually with surrounding dashes to create continuous separators

Design decisions made:
- Pipe separators (|) with single space padding on both sides for readability
- ANSI codes fully preserved in output but not counted toward width
- Padding applied to all cells (not just non-last column) for consistent alignment

No conflicts with Agent A's VisualWidth/PadToWidth functions:
- Both agents use same insertion point (after FormatDurationShort)
- Different function names, no overlaps
- FormatTable uses existing StripANSI directly (doesn't depend on VisualWidth)
