## Changes Made

### File: pkg/display/display.go
- Added `VisualWidth(s string) int` function (line 96-99)
  - Returns visual display width of string, ignoring ANSI escape codes
  - Uses existing StripANSI() function to strip codes
  - Counts runes (not bytes) for Unicode correctness

- Added `PadToWidth(s string, width int) string` function (line 101-108)
  - Right-pads string with spaces to reach target visual width
  - Preserves ANSI codes but doesn't count them toward width
  - Returns unchanged if already at or wider than target

### File: pkg/display/display_test.go
- Added `TestVisualWidth` test function (20 test cases covering):
  - Plain ASCII strings
  - ANSI color codes (should be ignored)
  - Unicode strings (Chinese, emoji, accented characters, Japanese)
  - Mixed ANSI + Unicode combinations

- Added `TestPadToWidth` test function (19 test cases covering):
  - Plain ASCII padding
  - ANSI-colored content preservation
  - Unicode string padding
  - Edge cases (already at width, wider than target)
  - Mixed ANSI and Unicode scenarios
