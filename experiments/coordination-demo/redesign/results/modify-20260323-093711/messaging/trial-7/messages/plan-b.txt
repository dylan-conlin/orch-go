PLAN-B: Unicode-Aware Truncate and TruncateWithPadding Refactoring
====================================================================

SCOPE:
- Modify: Truncate function (lines 14-19 in display.go)
- Modify: TruncateWithPadding function (lines 23-28 in display.go)
- Modify: TestTruncate test (lines 8-26 in display_test.go)
- Modify: TestTruncateWithPadding test (lines 28-47 in display_test.go)
- NO changes to: ShortID, StripANSI, FormatDuration, FormatDurationShort, or any other functions

CHANGES:

1. Truncate function:
   - Convert string to []rune for character counting
   - Check len(runes) instead of len(s) for initial check
   - For truncation, slice runes[:maxLen-3] then convert back to string
   - Append "..." suffix

2. TruncateWithPadding function:
   - Convert string to []rune for character counting
   - Check len(runes) > maxLen for truncation logic
   - For truncation: slice runes[:maxLen-3], convert to string, append "..."
   - For padding: convert string to runes, calculate padding = maxLen - len(runes)
   - Pad with spaces (ASCII)

3. Update TestTruncate:
   - Keep all existing test cases (they should pass unchanged)
   - Add test cases for:
     - Multi-byte UTF-8: "café" or emoji
     - Mixed ASCII and multi-byte
     - Exact maxLen match

4. Update TestTruncateWithPadding:
   - Keep all existing test cases (should pass unchanged)
   - Add test cases for:
     - Multi-byte UTF-8
     - Mixed ASCII and multi-byte
     - Exact maxLen match
   - Verify output length is still maxLen bytes (for ASCII tests) OR rune count (for new Unicode tests)

NO IMPACT ON:
- Other functions remain untouched
- ShortID, StripANSI, FormatDuration, FormatDurationShort unaffected
- No new dependencies
- No new exported functions
