## FormatDuration Refactoring Plan

### Functions to Modify
- FormatDuration (lines 49-82 in pkg/display/display.go)
- TestFormatDuration (lines 87-114 in pkg/display/display_test.go)

### Changes to FormatDuration

1. Refactor cascading if/else with threshold-based approach using slice of unit definitions
2. Add week support (7+ days → weeks)
3. Preserve all existing behavior for sub-week durations

#### Implementation Strategy
- Create a slice of unit definitions: {name, duration, nextSmaller}
- Iterate from largest to smallest unit (weeks → days → hours → minutes → seconds)
- Extract units and remainders
- Format output with at most 2 units (largest + next non-zero smaller)

#### Expected code changes
- Replace lines 49-82: Remove cascading if/else chains
- Use a structured approach with time thresholds
- Add week unit between days and months
- Preserve exact output format: "0s", "45s", "3m 12s", "2h 15m", "3d 5h", "1w", "1w 1d", etc.

### Changes to TestFormatDuration

- 7 * time.Hour * 24 → "1w"

- 14 * time.Hour * 24 → "2w"
- 15 * time.Hour * 24 + 12 * time.Hour → "2w 1d"
- 30 * time.Hour * 24 → "4w 2d"

### No Impact on Other Functions
