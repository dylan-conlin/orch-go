## Knowledge (What Was Learned)

### New Artifacts

### Decisions Made
- **Storage location:** `.orch/workspace/{agent}/screenshots/` because it follows existing workspace pattern, provides agent ownership, and enables automatic cleanup via workspace archival
- **Reference mechanism:** `screenshot-manifest.json` with metadata (filename, timestamp, context, tool, URL) because it enables verification without searching filesystem
- **Verification approach:** File-based verification (check files exist) in addition to current pattern-matching because it provides actual proof rather than claims

### Constraints Discovered
- MCP tools return base64 by design - agents must explicitly save to disk
- Glass CLI has no workspace awareness - needs convention or flag
- Can't break existing pattern-based verification - must be additive
