## Plain-Language Summary

The daemon's light auto-complete path (for `effort:small` agents) was failing 100% of the time because it skipped one interactive gate (explain-back) but left another (behavioral verification/gate2) that also requires a human. Since the daemon runs without a human, gate2 always failed. The fix: use `--headless` mode, which was designed exactly for non-interactive completions — it forces `review-tier=auto` (skipping all checkpoint gates) while preserving all other verification. This also means the brief is generated as part of the completion itself, making the separate headless brief call unnecessary.

## TLDR

Light auto-complete failed 100% because `CompleteLight` skipped gate1 but not gate2, both of which require a human. Fixed by using `--headless` mode which correctly skips all interactive checkpoint gates.

## Architectural Choices

### Use --headless instead of adding --review-tier auto
- **What I chose:** Switch CompleteLight to `--headless` mode entirely
- **What I rejected:** Adding `--review-tier auto` alongside existing `--skip-explain-back --skip-reason`
- **Why:** `--headless` is the correct abstraction for non-interactive daemon completions. It handles review-tier, explain-back skip, AND brief generation in one flag. Individual gate skips don't compose well — they address symptoms, not the structural need.
- **Risk accepted:** `--headless` also generates a brief as part of completion, making the separate `fireHeadlessCompletion` call redundant. Removed the call for cleanliness.
