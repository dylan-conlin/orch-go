     And on the Platform API, gpt-5.3-codex does NOT exist. It only goes up to gpt-5.2-codex.

     gpt-5.3-codex only exists on the Codex/ChatGPT backend — which requires OAuth
     authentication through the plugin. That's why it worked before (when OAuth tokens were
     present) and broke now.

     The Fix

     You need to re-authenticate with OpenAI OAuth:

     opencode auth login

     Choose the OpenAI / ChatGPT option, complete the OAuth flow, and this should restore the
     openai entry in auth.json. After that, gpt-5.3-codex requests will route through the Codex
     backend again instead of the Platform API.

     The gpt-5.3-codex model appeared in the models.dev cache and OpenCode lists it, but the
     plugin hasn't been updated to include it in its config. This might also need a plugin
     update — but first, restoring OAuth is the critical fix.
